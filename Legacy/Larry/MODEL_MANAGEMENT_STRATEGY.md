## 🤔 질문에 대한 답변

### Q1: 소형 데이터로 모델 만드는 건가요?

**A: 네, 맞습니다!**

**이유**:

1. **초기 프로토타입**: 먼저 작동하는 모델이 필요
2. **빠른 검증**: 소형 데이터로도 기본 동작 확인 가능
3. **점진적 개선**: 이후 더 많은 데이터로 개선

**현재 단계**:

```
소형 데이터 (113개) → 초기 모델 → 테스트 → 더 많은 데이터 수집 → 모델 개선
```

### Q2: 플레이할 때마다 모델을 계속 업그레이드해야 하나요?

**A: 자동 업그레이드는 비권장, 수동 업그레이드 권장**

**이유**:

- ❌ **자동 업그레이드 문제점**:

  - 훈련 시간이 오래 걸림 (수십 분~수시간)
  - 실시간 게임 중 훈련 불가능
  - 모델 품질 검증 필요
  - 데이터 품질 검증 필요

- ✅ **수동 업그레이드 권장**:
  - 주기적으로 데이터 수집 (예: 300장, 500장, 1000장)
  - 데이터 품질 검토 후 훈련
  - 모델 성능 평가 후 배포
  - 버전 관리 가능

## 📋 모델 업그레이드 전략

### 단계별 접근

#### Phase 1: 초기 모델 (현재)

- **데이터**: 113개 (소형)
- **목적**: 기본 동작 확인
- **Jeewon 작업**: epoch 50 훈련 중
- **결과**: `best.pt` 생성

#### Phase 2: 개선 모델 (1차)

- **데이터**: 300개 목표
- **목적**: 성능 향상
- **시점**: 초기 모델 테스트 후
- **방법**: 추가 게임 플레이 → 데이터 수집 → 재훈련

#### Phase 3: 최종 모델 (2차)

- **데이터**: 500-1000개
- **목적**: 최종 성능 최적화
- **시점**: 프로젝트 마감 전
- **방법**: 대량 데이터 수집 → 재훈련

### 업그레이드 프로세스

```
1. 데이터 수집 (게임 플레이)
   ↓
2. 데이터 품질 검토 (Jeewon)
   ↓
3. 모델 재훈련 (Jeewon)
   ↓
4. 성능 평가 (mAP, 정확도)
   ↓
5. 모델 배포 (GCP)
```

## 🗂️ 프로젝트 관리 방법

### 1. 모델 버전 관리

**디렉토리 구조**:

```
web_app/models/yolo/
├── v1_best.pt          # 초기 모델 (113개 데이터)
├── v2_best.pt          # 개선 모델 (300개 데이터)
├── v3_best.pt          # 최종 모델 (1000개 데이터)
└── best.pt             # 현재 사용 중인 모델 (심볼릭 링크 또는 복사본)
```

**버전 관리 전략**:

- 각 모델에 버전 번호 부여
- 성능 지표와 함께 문서화
- 이전 버전 보관 (롤백 가능)

### 2. 데이터 수집 계획

**단계별 목표**:

```
Phase 1: 113개 (완료) → 초기 모델
Phase 2: 300개 (목표) → 개선 모델
Phase 3: 500-1000개 (목표) → 최종 모델
```

**데이터 수집 방법**:

- Human Mode 플레이 (다양한 시나리오)
- AI Mode 플레이 (자동 수집)
- 다양한 난이도로 플레이

### 3. 모델 업그레이드 주기

**권장 주기**:

- **초기**: 즉시 (113개 → 모델 생성) ← **지금**
- **1차 개선**: 300개 수집 후 (1-2주 후)
- **2차 개선**: 500-1000개 수집 후 (프로젝트 마감 전)

**업그레이드 기준**:

- 데이터가 충분히 쌓였을 때 (300개 이상)
- 성능 개선이 필요할 때
- 프로젝트 마감 전 최종 버전

### 4. 협업 프로세스

**Jeewon (CV 담당)**:

1. 모델 훈련 및 평가
2. 모델 파일 관리 (`web_app/models/yolo/`)
3. 성능 지표 공유
4. 모델 업그레이드 제안

**Minsuk (배포 담당)**:

1. 데이터 수집 시스템 관리
2. 모델 배포 (GCP)

4. 배포 테스트

**Chloe (RL 담당)**:

1. RL 모델 훈련 (별도)
2. YOLO 모델과 협업 (Vision-based RL)

### 5. 문서화

**필요한 문서**:

- 모델 버전별 성능 지표
- 데이터셋 정보 (개수, 날짜)
